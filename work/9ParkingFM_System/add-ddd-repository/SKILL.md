# Add Repository Implementation

Generate an Infrastructure-layer Repository class from an existing Domain Repository interface. The interface was previously generated by `/9pfms:add-ddd-aggregate-efcore` and then customized by the user with additional method signatures.

## Prerequisites

Before this skill runs, the following should already exist (created by `/9pfms:add-ddd-aggregate-efcore`):
- The Aggregate Root class (e.g., `Menu : AggregateRoot<MenuId>`)
- The EF Core Configuration class
- The Domain Repository interface (e.g., `IMenuRepository : IRepository<Menu, MenuId>`)
- The `DbSet<T>` in `AppDbContext`

The user has then added custom method signatures (with optional comments) to the interface.

## Step 1: Gather context

Read these files to understand the full picture:

1. **The interface file** — the user will point you to it, or you find it in `Domain/Aggregates/{AggregateFolder}/`
2. **The corresponding Aggregate Root** — in the same folder, to understand entity structure, properties, Value Objects, and relationships
3. **AppDbContext.cs** — to find the correct `DbSet<T>` property names for cross-entity queries
4. **Existing repository implementations** — look in `Infrastructure/Persistence/Configuration/Domain/` for style reference

## Step 2: Refine the interface (if needed)

Review the interface and suggest improvements before implementing. Common refinements:

- Add `CancellationToken cancellationToken = default` if missing
- Suggest `Task<T?>` vs `Task<T>` based on whether "not found" is a valid case
- For methods returning collections, suggest `Task<List<T>>` or `Task<IReadOnlyList<T>>`

**Show the user your suggested refinements and wait for confirmation before proceeding.** If the interface looks good as-is, skip this step.

## Step 3: Generate the Repository implementation

### File location

Place the implementation in:
```
{InfrastructureProject}/Persistence/Configuration/Domain/{AggregateFolder}/{Name}Repository.cs
```

This co-locates the repository with the EF Core configuration, matching the project convention.

### Template

```csharp
using Microsoft.EntityFrameworkCore;
using {DomainNamespace};

namespace {InfrastructureNamespace}.Persistence.Configuration.Domain.{AggregateFolder};

public class {Name}Repository : Repository<{AggregateType}, {TId}>, I{Name}Repository
{
    private readonly AppDbContext _dbContext;

    public {Name}Repository(AppDbContext dbContext) : base(dbContext)
    {
        _dbContext = dbContext;
    }

    // ... custom method implementations only
    // Base CRUD (GetByIdAsync, AddAsync, Update, Delete) is inherited from Repository<T, TId>
}
```

### Key conventions

- **Inherit from `Repository<T, TId>`** — pass `AppDbContext` to `base(dbContext)`
- **Use `DbSet`** (from base class) for queries on the aggregate's own table
- **Use `_dbContext.{DbSetName}`** for cross-entity queries (e.g., join tables like `MenuActions`)
- Use `.Where().SingleOrDefaultAsync()` for single-entity lookups
- Use `.Where().ToListAsync()` for collection queries
- All async methods accept `CancellationToken cancellationToken = default`
- Value Object IDs can be compared directly with `==` in LINQ (EF Core handles conversion)

### Method implementation patterns

**Aggregate query (e.g., next sort order):**
```csharp
public async Task<int> GetNextSortOrderAsync(MenuId? parentId, CancellationToken cancellationToken = default)
{
    var maxSortOrder = await DbSet
        .Where(x => x.ParentId == parentId)
        .MaxAsync(x => (int?)x.SortOrder, cancellationToken);

    return (maxSortOrder ?? 0) + 1;
}
```

**Link via join table (upsert pattern):**
```csharp
public async Task LinkinActionAsync(MenuId menuId, ActionId actionId, CancellationToken cancellationToken = default)
{
    var existMenuAction = await _dbContext.MenuActions
        .Where(x => x.MenuId == menuId && x.ActionId == actionId)
        .SingleOrDefaultAsync(cancellationToken);

    if (existMenuAction is not null)
    {
        existMenuAction.Enable();
    }
    else
    {
        await _dbContext.MenuActions
            .AddAsync(Menu.LinkinAction(menuId, actionId, 0), cancellationToken);
    }
}
```

**Unlink via join table (soft delete):**
```csharp
public async Task UnlinkActionAsync(MenuId menuId, ActionId actionId, CancellationToken cancellationToken = default)
{
    var existMenuAction = await _dbContext.MenuActions
        .Where(x => x.MenuId == menuId && x.ActionId == actionId)
        .SingleOrDefaultAsync(cancellationToken);

    existMenuAction?.Disable();
}
```

**Query by foreign key:**
```csharp
public async Task<List<Order>> GetByCustomerIdAsync(int customerId, CancellationToken cancellationToken = default)
{
    return await DbSet
        .Where(x => x.CustomerId == customerId)
        .ToListAsync(cancellationToken);
}
```

**Get with related data (overriding base method to populate Ignored properties):**
```csharp
public override async Task<Menu?> GetByIdAsync(MenuId id, CancellationToken cancellationToken = default)
{
    var menu = await DbSet
        .FirstOrDefaultAsync(e => e.Id.Equals(id), cancellationToken);

    if (menu is null)
        return null;

    var childrenIds = await DbSet
        .Where(x => x.ParentId != null && x.ParentId.Equals(id))
        .Select(x => x.Id)
        .ToListAsync(cancellationToken);

    // menu.SetChildrenIds(childrenIds);

    return menu;
}
```

## Full Example

Given this interface at `Domain/Aggregates/MenuInfo/IMenuRepository.cs` (after user added custom methods):

```csharp
using YP.FinancialManagementSystem.Domain.Aggregates.ActionInfo;

namespace YP.FinancialManagementSystem.Domain.Aggregates.MenuInfo;

public interface IMenuRepository : IRepository<Menu, MenuId>
{
    Task<int> GetNextSortOrderAsync(MenuId? parentId, CancellationToken cancellationToken = default);
    Task LinkinActionAsync(MenuId menuId, ActionId actionId, CancellationToken cancellationToken = default);
    Task UnlinkActionAsync(MenuId menuId, ActionId actionId, CancellationToken cancellationToken = default);
}
```

The skill generates:

**`Infrastructure/Persistence/Configuration/Domain/MenuInfo/MenuRepository.cs`:**
```csharp
using Microsoft.EntityFrameworkCore;
using YP.FinancialManagementSystem.Domain.Aggregates.ActionInfo;
using YP.FinancialManagementSystem.Domain.Aggregates.MenuInfo;
using YP.FinancialManagementSystem.Infrastructure.Shared;

namespace YP.FinancialManagementSystem.Infrastructure.Persistence.Configuration.Domain.MenuInfo;

public class MenuRepository : Repository<Menu, MenuId>, IMenuRepository
{
    private readonly AppDbContext _dbContext;

    public MenuRepository(AppDbContext dbContext) : base(dbContext)
    {
        _dbContext = dbContext;
    }

    public async Task<int> GetNextSortOrderAsync(MenuId? parentId, CancellationToken cancellationToken = default)
    {
        var maxSortOrder = await DbSet
            .Where(x => x.ParentId == parentId)
            .MaxAsync(x => (int?)x.SortOrder, cancellationToken);

        return (maxSortOrder ?? 0) + 1;
    }

    public async Task LinkinActionAsync(MenuId menuId, ActionId actionId, CancellationToken cancellationToken = default)
    {
        var existMenuAction = await _dbContext.MenuActions
            .Where(x => x.MenuId == menuId && x.ActionId == actionId)
            .SingleOrDefaultAsync(cancellationToken);

        if (existMenuAction is not null)
        {
            existMenuAction.Enable();
        }
        else
        {
            await _dbContext.MenuActions
                .AddAsync(Menu.LinkinAction(menuId, actionId, 0), cancellationToken);
        }
    }

    public async Task UnlinkActionAsync(MenuId menuId, ActionId actionId, CancellationToken cancellationToken = default)
    {
        var existMenuAction = await _dbContext.MenuActions
            .Where(x => x.MenuId == menuId && x.ActionId == actionId)
            .SingleOrDefaultAsync(cancellationToken);

        existMenuAction?.Disable();
    }
}
```
